# سياسة الأمان

Asas OS مشروع تجريبي وغير مناسب لحماية بيانات أو أجهزة إنتاجية.

## الإصدارات المدعومة

تُقبل إصلاحات الأمان للفرع التطويري الافتراضي ولآخر فرع إصدار عام إذا كان موجودًا.

## الإبلاغ عن ثغرة

من فضلك لا تنشر تفاصيل استغلال في Issue عامة قبل أن يحصل المشرفون على فرصة
لمراجعة البلاغ.

إذا كانت قناة security خاصة مفعلة على GitHub، استخدمها. إذا لم تكن موجودة،
افتح Issue قصيرة تطلب قناة تواصل خاصة بدون وضع تفاصيل حساسة.

البلاغ الجيد يتضمن:

- commit أو release المتأثر.
- البيئة: QEMU أو Hyper-V أو جهاز فعلي.
- خطوات إعادة الإنتاج.
- هل المشكلة تخص disk writes أو filesystem metadata أو boot أو privilege
  boundaries أو memory safety.
- السلوك المتوقع والسلوك الفعلي.

## مناطق عالية الحساسية

تعامل مع هذه المناطق كحساسة أمنيًا:

- bootloader وتحميل kernel image.
- paging وmemory allocation وحدود user/kernel.
- syscall validation.
- الكتابة على الأقراص وتعديل partitions وإصلاح filesystems.
- تحليل virtual disk formats.
- تحليل network packets.

## الإفصاح

سيراجع المشرفون البلاغات حسب الوقت المتاح. لأن المشروع تجريبي، لا يوجد وعد
زمني ثابت للاستجابة حتى الآن.
