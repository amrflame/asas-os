# Code of Conduct

Asas OS should be a calm, technical, and welcoming place to build an operating
system together. The project is ambitious, but the community should feel
approachable: questions, experiments, careful bug reports, and first patches all
belong here.

## Expected Behavior

- Be respectful and patient.
- Assume good intent, especially with new contributors.
- Critique code and ideas, not people.
- Explain tradeoffs clearly when reviewing complex kernel or filesystem work.
- Help keep discussions focused on the project.
- Encourage learning while still protecting users from unsafe technical choices.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discrimination based on identity, background, language, or experience level.
- Publishing private information without permission.
- Repeatedly derailing technical discussions.
- Pressuring maintainers to merge unsafe storage, filesystem, or security changes.

## Enforcement

Maintainers may edit or delete comments, close issues, reject pull requests, or
block participants when behavior harms the project community.

If you need to report a conduct issue, contact the maintainers privately. If no
private contact is listed yet, open a minimal issue asking for a private contact
channel without posting sensitive details.
